# Room 前端方案

## 目标

Room 前端第一版服务于移动端优先的房间聊天流程，覆盖注册、登录、房间列表、创建/进入房间、房间详情、文字消息、麦克风状态和 WebSocket 房间事件同步。

第一版目标是把主链路做稳，不追求复杂社交能力、离线消息、多端状态同步或房间内复杂音视频能力。

## 技术选型

- 框架：uni-app / Vue 3
- 语言：JavaScript 或 TypeScript，项目初始化时统一决定
- 状态管理：优先使用轻量 store，保存 token、当前用户、当前房间和 WebSocket 状态
- 网络请求：统一封装 request，处理业务响应结构 `code/message/data`
- WebSocket：封装 room socket client，页面只订阅领域事件
- 样式：移动端优先，保留扩展到 H5 的结构

## 目录规划

```txt
room-app
  docs
    前端方案.md
  api
  components
  pages
  store
  utils
  static
```

建议后续按实际 uni-app 初始化结果调整，但保持以下边界：

- `api` 只放 HTTP 接口封装，不写页面逻辑。
- `store` 管理登录态、用户资料、当前房间状态。
- `utils/request` 统一加 token、解析响应、处理错误提示。
- `utils/ws` 统一管理 WebSocket 连接、重连策略和事件分发。
- `pages` 只负责页面交互和渲染。

## 页面范围

第一版页面：

- 登录页
- 注册页
- 找回密码页
- 当前用户资料页
- 房间列表页
- 房间详情页
- 头像上传流程

房间详情页包含：

- 房间基础信息
- 成员列表
- 房主标识
- 麦克风开关状态
- 历史消息列表
- 文字消息发送
- WebSocket 实时事件更新

## API 对接

后端统一响应：

```json
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```

前端约定：

- HTTP 状态码不是主要判断依据，业务成功以 `code === 200` 为准。
- `Authorization` 使用 `Bearer ${token}`。
- token 持久化到本地存储，启动时恢复登录态。
- 业务错误统一弹出 `message`，必要时页面自行补充上下文。

## WebSocket 约定

连接地址：

```txt
/api/v1/ws/rooms/:room_id
```

请求头：

```txt
Authorization: Bearer jwt_token
```

第一版事件：

- `member.joined`
- `member.left`
- `room.owner_changed`
- `message.created`
- `member.mic_updated`

前端处理原则：

- 进入房间成功后再建立 WebSocket。
- WebSocket 断开按后端语义等价于离开房间。
- 收到 `message.created` 后追加消息。
- 收到成员事件后更新成员列表和 `current_members`。
- 收到 `room.owner_changed` 后更新 `room.owner_id`，成员 `is_owner` 以后端返回为准。

## 状态模型

建议 store 拆分：

- `authStore`：token、登录、登出、恢复登录态
- `userStore`：当前用户资料、更新昵称、更新头像
- `roomStore`：房间列表、当前房间详情、成员、消息、麦克风状态
- `socketStore` 或 `roomSocket`：当前房间 WebSocket 连接状态和事件分发

关键约束：

- 当前用户同一时间只能在一个房间。
- 房主来源以后端 `room.owner_id` 为准。
- 成员对象里的 `is_owner` 是后端返回的派生字段，前端只用于展示。
- 断开 WebSocket 后需要回到房间列表或重新同步用户当前房间状态。

## 第一版不做

- 不做多房间同时在线。
- 不做断线后自动保留房间席位。
- 不做消息客户端幂等 ID。
- 不做离线消息推送。
- 不做复杂音视频能力。
- 不做前端全局权限系统，只处理登录态。

## 开发顺序

1. 初始化项目结构和基础样式。
2. 封装 request、token 存储和 API 模块。
3. 完成注册、登录、找回密码、头像上传。
4. 完成房间列表、创建房间、进入房间。
5. 完成房间详情、成员列表、历史消息、发送消息。
6. 接入 WebSocket 事件。
7. 做移动端真机调试和接口联调。

## 联调依赖

- 后端服务地址，例如 `http://127.0.0.1:9999`。
- 后端数据库已执行最新 migration。
- WebSocket 客户端需要支持自定义 `Authorization` 请求头。
- 上传头像依赖腾讯云 COS 配置可用。
