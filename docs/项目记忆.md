# Room Android Project Memory

## Current Status

- This repository contains Android client planning docs and an Android Studio `Empty Activity` Compose project.
- Backend project is expected to be a sibling directory: `../room-api`.
- Backend first version is implemented and usable for Android integration.
- Backend server uses fixed port `9999`.
- App docs are intentionally concise and should stay close to backend docs style.
- Do not add "第一版不做" or negative scope lists; docs should describe what will be built.
- Android package name is `love.moonc.room`.
- Android module is `app`.
- `minSdk` is `26`.
- Gradle uses Version Catalog in `gradle/libs.versions.toml`; keep dependency versions centralized there.
- `.gitignore` is user-managed. Do not change it unless the user explicitly asks.

- Android emulator backend base URL:

```txt
http://10.0.2.2:9999/
```

- Real-device backend base URL is a Debug configuration value, normally `http://<backend-lan-ip>:9999/` when phone and backend machine are on the same LAN.
- Debug backend base URL is read from `local.properties` key `ROOM_BACKEND_BASE_URL` when present.
- If `ROOM_BACKEND_BASE_URL` is absent, Debug uses `http://10.0.2.2:9999/`.
- `local.properties` stays local and must not be committed.

## Important Product Decisions

- Use simple default Material 3 UI. Do not make complex custom visual design.
- Use two bottom Tabbar pages after login:
  - `首页`: room list.
  - `我的`: current user profile.
- Separate pages:
  - `创建房间`.
  - `编辑资料`.
  - `房间详情`.
- `创建房间` and `编辑资料` use a top-left back arrow.
- Room detail has no bottom Tabbar and no profile entry.
- Room list top-right entry is enough for creating a room.
- Time should display year, month, day, hour, minute and second.

## Naming

- UI text uses `账号`.
- API, database and app data models use `account`.
- Do not use `username` / `用户名` for this project.
- Nickname remains `nickname` / `昵称`.

## Auth And User

- Login and register success should use the returned `user` directly for navigation and local state.
- App startup with existing token calls `/api/v1/users/me`.
- `/api/v1/users/me` is used to validate token and restore current user.
- Register avatar is selected by tapping the avatar circle.
- Avatar source menu contains `相册` and `拍照`.
- Edit profile opens a separate page and supports changing nickname and avatar.
- Camera support is required.

## Room And Chat

- Create room page supports `2人房` and `8人房`.
- Default selected room size is `8人房`.
- Full rooms are disabled or clearly shown as full.
- Current user mic control uses Material `Switch`.
- Room detail has `加载更早消息`.
- WebSocket abnormal disconnect returns to main home and shows:

```txt
连接已断开，已离开房间
```
