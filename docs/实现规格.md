# Room Android 实现规格

本文描述 Room Android 客户端的实现边界、模块划分、接口对接和页面要求。页面流转和状态流转见 `docs/客户端逻辑设计.md`，后端接口以 `room-api/docs/接口文档.md` 为准。

## 项目配置

- Android 单模块项目，模块名 `app`。
- 包名和 namespace：`love.moonc.room`。
- `minSdk = 26`。
- 语言：Kotlin。
- UI：Jetpack Compose + Material 3。
- 架构：Compose + Hilt ViewModel + StateFlow。
- 网络：Retrofit + OkHttp + kotlinx.serialization。
- 图片加载：Coil Compose。
- 本地存储：DataStore Preferences。
- WebSocket：OkHttp WebSocket。
- 导航：Navigation Compose。
- Gradle、Android Gradle Plugin、Kotlin、Compose、Hilt 等版本由实现时选择当前稳定版本。
- Debug 后端地址通过配置项提供。
- 模拟器默认后端地址：`http://10.0.2.2:9999/`。
- 真机调试后端地址使用同一局域网地址，例如 `http://<backend-lan-ip>:9999/`。
- WebSocket 地址从 HTTP base url 派生，例如 `http://10.0.2.2:9999/` 派生为 `ws://10.0.2.2:9999/api/v1/ws/rooms/{roomId}`。

## 依赖

Gradle 使用 Android Gradle Plugin、Kotlin Android、Kotlin Serialization、Hilt 插件。依赖包含：

- AndroidX Core、Activity Compose、Lifecycle Runtime Compose、Lifecycle ViewModel Compose。
- Compose BOM、Compose UI、Material 3、Compose tooling preview/debug tooling。
- Navigation Compose。
- Hilt Android、Hilt Navigation Compose。
- DataStore Preferences。
- Retrofit、OkHttp、OkHttp Logging Interceptor。
- kotlinx.serialization JSON、Retrofit kotlinx.serialization converter。
- Coil Compose。

Debug 包启用 OkHttp Logging Interceptor；release 包关闭请求/响应日志。


边界：

- `data/api` 放 Retrofit API 定义。
- `data/model` 放后端请求和响应模型。
- `data/storage` 放 token 持久化。
- `data/websocket` 放房间 WebSocket 连接、事件解析和关闭。
- `core/network` 放统一响应、token interceptor、网络错误转换。
- `ui/*` 放 Compose 页面和 Hilt ViewModel。
- 第一版 ViewModel 可以直接注入 `RoomApi`、`TokenStore`、`RoomSocketFactory`。

## 网络规则

后端 HTTP 响应统一为：

```json
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```

客户端规则：

- HTTP 状态码不是主要判断依据，业务成功以 `code == 200` 为准。
- 业务失败优先展示后端 `message`。
- `code == 401` 或 `/users/me` 恢复失败时清空 token 并回登录页。
- 网络失败展示通用网络错误。
- JSON 使用 `ignoreUnknownKeys = true`。
- 请求和响应字段按后端 snake_case 命名，Kotlin 侧使用 kotlinx.serialization 显式映射。
- 时间字段先按 ISO 8601 字符串保存，UI 展示时再格式化。

## Token

- DataStore Preferences 文件名：`room_settings`。
- token key：`auth_token`。
- 登录和注册成功后保存 token。
- 退出登录时清空 token。
- OkHttp Interceptor 自动追加 `Authorization: Bearer {token}`。
- App 启动时如果本地有 token，必须请求 `/api/v1/users/me` 恢复登录态。

## HTTP API

Retrofit API 覆盖以下接口：

| 功能 | 方法 | 路径 | 鉴权 |
| --- | --- | --- | --- |
| 上传注册头像 | POST multipart | `/api/v1/uploads/avatar` | 否 |
| 发送注册验证码 | POST | `/api/v1/auth/register-code` | 否 |
| 注册 | POST | `/api/v1/auth/register` | 否 |
| 登录 | POST | `/api/v1/auth/login` | 否 |
| 发送重置密码验证码 | POST | `/api/v1/auth/password-reset-code` | 否 |
| 重置密码 | POST | `/api/v1/auth/reset-password` | 否 |
| 当前用户 | GET | `/api/v1/users/me` | 是 |
| 更新昵称 | PATCH | `/api/v1/users/me` | 是 |
| 更新头像 | POST multipart | `/api/v1/users/me/avatar` | 是 |
| 房间列表 | GET | `/api/v1/rooms?page=1&page_size=20` | 是 |
| 创建房间 | POST | `/api/v1/rooms` | 是 |
| 房间详情 | GET | `/api/v1/rooms/:room_id` | 是 |
| 进入房间 | POST | `/api/v1/rooms/:room_id/join` | 是 |
| 离开房间 | POST | `/api/v1/rooms/:room_id/leave` | 是 |
| 更新麦克风 | PATCH | `/api/v1/rooms/:room_id/mic` | 是 |
| 历史消息 | GET | `/api/v1/rooms/:room_id/messages?limit=20&before_id=100` | 是 |
| 发送消息 | POST | `/api/v1/rooms/:room_id/messages` | 是 |

multipart 上传字段名固定为 `file`。

## 数据模型

客户端模型与后端对象一一对应。

### User

- `id`
- `account`
- `email`
- `nickname`
- `avatar_url`
- `current_room_id`
- `created_at`
- `updated_at`

`current_room_id` 为 `null` 表示用户当前不在房间内。

### Room

- `id`
- `name`
- `owner_id`
- `current_members`
- `max_members`
- `created_at`

房主以 `owner_id` 为准。

### RoomMember

- `user_id`
- `nickname`
- `avatar_url`
- `is_owner`
- `mic_status`
- `joined_at`

`mic_status` 取值为 `on` 或 `off`。

### Message

- `id`
- `room_id`
- `sender_id`
- `sender_nickname`
- `sender_avatar_url`
- `type`
- `content`
- `created_at`

第一版消息类型为 `text`。

### 主要请求

- 注册：`account`、`email`、`email_code`、`password`、`nickname`、`avatar_url`。
- 登录：`account`、`password`。
- 重置密码：`email`、`email_code`、`new_password`。
- 创建房间：`max_members`，只能是 `2` 或 `8`。
- 更新麦克风：`mic_status`，只能是 `on` 或 `off`。
- 发送消息：`content`。

## WebSocket

连接：

- 路径：`/api/v1/ws/rooms/:room_id`。
- 请求头：`Authorization: Bearer {token}`。
- 进入房间成功后再建立 WebSocket。
- 主动离开时先调用离开接口，再关闭 WebSocket。
- WebSocket 异常断开时，按后端语义视为已离开房间，回主界面首页并提示“连接已断开，已离开房间”。

事件外层：

- `type`
- `room_id`
- `data`

事件类型：

| type | data |
| --- | --- |
| `member.joined` | `member`、`current_members` |
| `member.left` | `user_id`、`current_members` |
| `room.owner_changed` | `owner_id`、`members` |
| `message.created` | `message` |
| `member.mic_updated` | `member` |

事件处理：

- `member.joined`：按 `user_id` 追加或覆盖成员，并更新人数。
- `member.left`：按 `user_id` 删除成员，并更新人数。
- `room.owner_changed`：更新 `owner_id`，事件携带 `members` 时覆盖成员列表。
- `message.created`：按消息 `id` 去重后追加。
- `member.mic_updated`：按 `user_id` 替换成员。

## 导航

页面路由：

- `splash`
- `login`
- `register`
- `reset-password`
- `main/home`
- `main/me`
- `profile/edit`
- `rooms/create`
- `rooms/{roomId}`

登录后的主界面使用底部 Tabbar：

- `首页`：房间列表。
- `我的`：用户资料。

启动逻辑：

- 无 token：进入登录页。
- 有 token：请求 `/users/me`。
- `/users/me` 成功且 `current_room_id == null`：进入主界面首页。
- `/users/me` 成功且 `current_room_id != null`：进入房间详情。
- `/users/me` 失败：清空 token，进入登录页。
- 登录和注册成功后直接使用接口返回的 `user` 导航，不额外请求 `/users/me`。

## ViewModel

### AuthViewModel

负责：

- 登录。
- 发送注册验证码。
- 注册。
- 发送重置密码验证码。
- 重置密码。
- 注册头像上传。
- 保存 token。

本地校验：

- 账号符合 `^[A-Za-z0-9_]{4,20}$`。
- 密码 6-20 位。
- 昵称 trim 后 1-8 个字符。
- 邮箱非空且符合基本邮箱格式。
- 验证码非空。
- 注册必须已有头像 URL。

### ProfileViewModel

负责：

- 加载当前用户。
- 更新昵称。
- 更新头像。
- 退出登录。

### RoomListViewModel

负责：

- 加载房间第一页。
- 下拉刷新。
- 加载更多。
- 进入房间。

分页：

- 默认 `page = 1`，`page_size = 20`。
- `hasMore` 根据已加载数量和后端 `total` 判断。

### RoomDetailViewModel

负责：

- 加载房间详情。
- 加载历史消息。
- 建立和关闭 WebSocket。
- 发送文字消息。
- 加载更早消息。
- 更新麦克风状态。
- 主动离开房间。
- 处理 WebSocket 事件。

进入房间详情顺序：

1. 请求房间详情。
2. 请求历史消息。
3. 建立 WebSocket。

消息规则：

- 输入 trim 后 1-50 字才允许发送。
- 发送成功后清空输入框。
- 消息列表等待 `message.created` 广播追加。
- 加载更早消息使用当前最小 message id 作为 `before_id`。

## 头像文件处理

- 头像来源支持相册选择和拍照。
- 相册可使用 `PickVisualMedia` 或 `GetContent("image/*")`。
- 拍照可使用 `TakePicture`。
- 拍照需要 FileProvider，文件放 app cache/images 目录。
- 文件类型限制为 JPEG、PNG、WebP。
- 本地限制文件大小不超过 2MB。
- MIME 类型优先使用 ContentResolver，fallback 为 `image/jpeg`。
- 上传成功后展示头像预览。
